# fwupd Rust Bindings

Rust crates for writing fwupd firmware update plugins.

## Crate Structure

```text
rust/
├── fwupd-sys/           # FFI bindings for libfwupd (GIR-generated)
├── fwupdplugin-sys/     # FFI bindings for libfwupdplugin (GIR-generated)
├── xmlb-sys/            # FFI bindings for libxmlb (GIR-generated)
├── fwupdplugin/         # Safe Rust API for plugin development
└── examples/
    └── fu-plugin-example/   # Example plugin demonstrating the API
```

## Quick Start

### Prerequisites

- Rust toolchain (rustc, cargo)
- fwupd built from source (for headers and pkg-config files)

### Setup

```sh
# Build fwupd first (see top-level README)
meson setup build
ninja -C build

# Build the Rust crates (done automatically if cargo is available)
# Or manually:
PKG_CONFIG_PATH=$(pwd)/build/meson-uninstalled cargo build \
    --manifest-path rust/Cargo.toml
```

### Writing a Plugin

Create a new Rust crate with `crate-type = ["cdylib"]`:

```toml
# Cargo.toml
[package]
name = "fu-plugin-my-device"
version = "0.1.0"
edition = "2021"

[lib]
crate-type = ["cdylib"]
name = "fu_plugin_my_device"

[dependencies]
fwupdplugin = { path = "../fwupdplugin" }
glib = "0.20"
```

Implement the plugin:

```rust
// src/lib.rs
use glib::prelude::*;
use fwupdplugin::prelude::*;

// Define a custom device type
mod device_imp {
    use glib::subclass::prelude::*;
    use fwupdplugin::prelude::*;

    #[derive(Default)]
    pub struct MyDevice;

    #[glib::object_subclass]
    impl ObjectSubclass for MyDevice {
        const NAME: &'static str = "FuMyDevice";
        type Type = super::MyDevice;
        type ParentType = fwupdplugin::UsbDevice;
    }

    impl ObjectImpl for MyDevice {}
    impl DeviceImpl for MyDevice {
        fn probe(&self) -> Result<(), glib::Error> {
            let dev = self.obj();
            dev.set_name("My Device");
            dev.set_vendor("ACME Corp");
            Ok(())
        }
        fn write_firmware(
            &self,
            _firmware: &Firmware,
            _progress: &Progress,
            _flags: fwupdplugin::fwupd_sys::FwupdInstallFlags,
        ) -> Result<(), glib::Error> {
            // Write firmware to device hardware
            Ok(())
        }
    }
    impl UdevDeviceImpl for MyDevice {}
    impl UsbDeviceImpl for MyDevice {}
}

glib::wrapper! {
    pub struct MyDevice(ObjectSubclass<device_imp::MyDevice>)
        @extends fwupdplugin::UsbDevice,
                 fwupdplugin::UdevDevice,
                 fwupdplugin::Device;
}

// Define the plugin
#[derive(Default)]
struct MyPlugin;

impl PluginImpl for MyPlugin {
    fn constructed(&self, plugin: &Plugin) {
        plugin.add_udev_subsystem("usb");
        plugin.add_device_gtype(MyDevice::static_type());
    }
}

fwupdplugin::export_plugin!(MyPlugin);
```

### Installing a Plugin

```sh
# Build the plugin
PKG_CONFIG_PATH=<fwupd-builddir>/meson-uninstalled cargo build --release

# The output is target/release/libfu_plugin_my_device.so
# Copy it to the fwupd plugin directory
PLUGIN_DIR=$(PKG_CONFIG_PATH=... pkg-config --variable=libdir_pkg fwupdplugin)
sudo cp target/release/libfu_plugin_my_device.so "$PLUGIN_DIR/"
```

## Regenerating FFI Bindings

The `-sys` crates are generated by the [gir](https://github.com/gtk-rs/gir)
tool from GObject Introspection `.gir` files:

```sh
# Install gir
cargo install --git https://github.com/gtk-rs/gir

# Copy fwupd GIR files from the build directory
mkdir -p rust/gir-files
cp build/libfwupd/Fwupd-2.0.gir \
   build/libfwupdplugin/FwupdPlugin-1.0.gir \
   rust/gir-files/

# Regenerate (from each sys crate directory)
cd rust/fwupdplugin-sys
rm Cargo.toml Cargo.lock
gir -o . -m sys
# Then manually fix: Cargo.toml dependencies, lib.rs FuDeviceClass
```

## Architecture

### Plugin Loading

fwupd loads modular plugins by:

1. Scanning the plugin directory for `libfu_plugin_*.so` files
2. Opening each with `g_module_open()`
3. Looking up `fu_plugin_init_vfuncs` via `g_module_symbol()`
4. Calling it with a `FuPluginVfuncs` pointer to populate

The `export_plugin!` macro generates step 3-4, wiring Rust `PluginImpl`
trait methods to the C vtable via trampoline functions.

### Device Subclassing

Devices use `glib::subclass` to create GObject subclasses from Rust.
The inheritance chain is:

```text
GObject -> FwupdDevice -> FuDevice -> FuUdevDevice -> FuUsbDevice -> FuHidDevice
```

The `DeviceImpl` trait provides the virtual methods (probe, setup,
open, close, write_firmware, etc.) that plugin authors override.
